# AI自律画像生成 — harness-harness適用指針

## 概要

本ドキュメントは、AI自律画像生成の研究知見をharness-harnessの各機能にどう反映するかの指針を定める。findings.md の推奨構成を、プロジェクトハーネスの作成・診断・改善に自動的に適用できるようにする。

---

## 1. create-harness: 新プロジェクトに画像生成MCP設定を自動提案

### 適用タイミング

create-harnessスキル実行時、プロジェクトの性質を分析して画像生成ニーズを検出し、MCP設定を自動提案する。

### 検出ロジック

以下のシグナルから画像生成ニーズを判定:

| シグナル | 検出方法 | 信頼度 |
|---------|---------|--------|
| コンテンツ制作プロジェクト | package.jsonやREADMEに「blog」「note」「article」「content」等のキーワード | 高 |
| 既存の画像ディレクトリ | `images/`, `assets/img/`, `thumbnails/` 等のディレクトリ存在 | 高 |
| マークダウンファイルの画像参照 | `![` パターンの多用 | 中 |
| SNS関連のコード・設定 | Twitter API, note API, OGP設定等 | 高 |
| ユーザーの明示的要求 | 「画像生成も入れて」等の発言 | 最高 |

### 提案内容

画像生成ニーズが検出された場合、以下をCLAUDE.mdおよびsettings.jsonに追加提案:

#### CLAUDE.md追記案

```markdown
## 画像生成

本プロジェクトでは画像生成MCPを使用してサムネイル・差込画像を自動生成できる。

### 利用可能なMCPツール
- `mcp-image`: Gemini NB2/Pro による画像生成（推奨）
- 直接API: Imagen 4, GPT Image 1.5, FLUX.2 等

### 画像生成時の方針
- 用途に応じてモデルを使い分ける（kb/research-themes/ai-image-generation/findings.md参照）
- 生成画像はClaude Visionで品質チェックしてから使用
- AI生成画像にはその旨を明記（法規制対応）
```

#### settings.json MCP追記案

```json
{
  "mcpServers": {
    "mcp-image": {
      "command": "npx",
      "args": ["-y", "mcp-image"],
      "env": {
        "GEMINI_API_KEY": "${GEMINI_API_KEY}"
      }
    }
  }
}
```

### 段階的開示

- 検出スコアが低い場合: 「画像生成MCPも設定できますが、必要ですか？」と確認
- 検出スコアが高い場合: 設定を提案に含め、不要なら除外してくれとコメント付き

## 2. init-project: Phase 0.5で画像生成ニーズを検出し、MCP設定を注入

### Phase 0.5の位置づけ

init-projectスキルの既存フローに「Phase 0.5: メディア・画像生成ニーズ分析」を追加。

```
Phase 0: プロジェクト種別判定
Phase 0.5: メディア・画像生成ニーズ分析  ← NEW
Phase 1: ハーネス構造生成
Phase 2: MCP/ツール設定
Phase 3: カスタマイズ・最終調整
```

### Phase 0.5の処理フロー

1. **ニーズ検出**: プロジェクトの目的・説明から画像生成が有益かを判定
2. **用途分類**: サムネ量産 / 文字入り画像 / フォトリアル / 差込画像 のどれに該当するか
3. **推奨モデル選定**: findings.md の用途別推奨テーブルから最適なモデルを選択
4. **MCP設定生成**: 推奨モデルに対応するMCP設定を生成
5. **コスト見積**: 想定月間生成量からコスト見積を提示

### 判定マトリクス

| プロジェクト種別 | 画像生成ニーズ | 推奨MCP | 推奨モデル |
|----------------|-------------|---------|----------|
| ブログ/note/記事系 | 高（サムネ必須） | mcp-image | Imagen 4 Fast/Standard |
| SNS自動投稿系 | 高（OGP画像） | mcp-image | GPT Image 1.5 Low/Med |
| ドキュメント/技術文書系 | 低（図表はコードで生成） | 提案のみ | - |
| Webアプリ/SaaS系 | 中（マーケティング素材） | mcp-imagenate | マルチプロバイダ |
| CLI/ライブラリ系 | 低（READMEバッジ程度） | 不要 | - |
| データ分析系 | 低（グラフはmatplotlib等） | 不要 | - |

## 3. diagnose-harness: 既存プロジェクトの画像生成設定を診断

### 診断チェックリスト

既存プロジェクトのハーネスを診断する際、以下の画像生成関連チェックを追加。

| チェック項目 | 期待状態 | 診断アクション |
|-------------|---------|-------------|
| 画像生成MCPの有無 | コンテンツ系プロジェクトにMCP設定あり | 未設定なら提案 |
| DALL-E 3 使用の有無 | 未使用（2026/5/12終了） | 使用中なら移行警告 |
| API Key管理 | 環境変数経由、ハードコードなし | ハードコードなら警告 |
| 画像生成プロンプトの品質 | モデル別最適化されている | 汎用プロンプトなら改善提案 |
| コスト最適化 | 用途に対して適切なモデル/品質 | 過剰品質ならダウングレード提案 |
| 法規制対応 | AI生成明記あり、透かし保持 | 未対応なら警告 |

### 診断レポートの画像生成セクション

```markdown
## 画像生成設定診断

### 現状
- [ ] 画像生成MCP: {設定あり/なし}
- [ ] 使用モデル: {モデル名}
- [ ] コスト効率: {適切/要改善}
- [ ] DALL-E 3依存: {あり/なし}

### 改善提案
- [A/B/C/D] {提案内容}
```

### 優先度基準

| 優先度 | 条件 | 例 |
|--------|------|-----|
| A (必須) | セキュリティ・規約違反 | API Keyハードコード、DALL-E 3終了未対応 |
| B (推奨) | コスト最適化・品質向上 | 用途に対して過剰品質、モデル未最適化 |
| C (任意) | 機能追加 | 画像生成MCP追加、自動評価パイプライン |
| D (参考) | 将来的な改善 | Agent SDK multi-agent化、Batch API対応 |

## 4. cross-project-copy: mcp-image設定を他プロジェクトに展開

### コピー対象

| 設定種別 | コピー元 | コピー先条件 | カスタマイズ |
|---------|---------|------------|-----------|
| MCP設定（mcp-image） | 画像生成導入済みプロジェクト | コンテンツ系プロジェクト | API Key は各プロジェクト固有 |
| CLAUDE.md画像生成セクション | テンプレート | 画像生成MCP導入プロジェクト | プロジェクト固有の用途を反映 |
| プロンプトテンプレート | 実績のあるプロジェクト | 同種のコンテンツプロジェクト | ブランド・トーン調整 |
| 評価基準（Claude Vision用） | benchmark-design.md | 画像生成導入プロジェクト | 媒体固有の基準調整 |

### コピーフロー

```
1. 展開元プロジェクトの画像生成設定を収集
2. 展開先プロジェクトの種別・ニーズを判定
3. 設定をプロジェクト固有にカスタマイズ
4. 差分をユーザーに提示、承認後にコピー
5. 動作確認（テスト画像1枚生成）
```

### 注意事項

- API Key は絶対にコピーしない。環境変数参照 `${GEMINI_API_KEY}` の形式で統一
- ブランドガイドライン（色、フォント、トーン）はプロジェクト固有。汎用テンプレートのみコピー
- コスト見積は展開先の想定利用量で再計算して提示

## 5. 対象プロジェクト: 記事サムネ/差込画像の自動生成パイプライン

### パイプライン概要

対象プロジェクトプロジェクトにおいて、記事作成フローに画像自動生成を統合する。

```
[記事テーマ決定]
      |
      v
[記事本文生成] ──→ [差込画像生成]
      |                    |
      v                    v
[サムネイル生成] ──→ [品質チェック (Claude Vision)]
      |                    |
      v                    v
[記事公開パッケージ完成]
```

### サムネイル生成パイプライン

| ステップ | 処理 | ツール/モデル |
|---------|------|-------------|
| 1. テーマ分析 | 記事タイトル・本文から画像要件を抽出 | Claude (テキスト分析) |
| 2. プロンプト生成 | 要件からモデル別最適化プロンプト生成 | Claude (Prompt Composer) |
| 3. 画像生成 | プロンプトで画像生成（2-3枚候補） | mcp-image or Imagen 4 API |
| 4. 品質評価 | 候補画像をClaude Visionで評価・選定 | Claude Vision (Critic) |
| 5. 後処理 | リサイズ(1280x670)、WebP変換、圧縮 | ImageMagick / sharp |
| 6. 配置 | 記事パッケージに画像を配置 | ファイル操作 |

### 差込画像生成パイプライン

| ステップ | 処理 | ツール/モデル |
|---------|------|-------------|
| 1. 挿入点検出 | 記事本文の適切な画像挿入ポイントを検出 | Claude (テキスト分析) |
| 2. コンテキスト分析 | 前後の段落から画像の要件を推論 | Claude |
| 3. 画像生成 | コンテキストに合った画像を生成 | mcp-image (standard品質) |
| 4. 品質評価 | 記事の文脈との整合性を評価 | Claude Vision |
| 5. 配置 | マークダウンに画像参照を挿入 | テキスト編集 |

### 対象プロジェクト固有の設定

```yaml
# 対象プロジェクト CLAUDE.md に追記する内容
image_generation:
  thumbnail:
    model: imagen-4-standard  # 標準品質
    fallback: gpt-image-1.5-medium
    size: 1280x670
    format: webp
    max_candidates: 3
  inline:
    model: imagen-4-fast  # コスパ重視
    fallback: gpt-image-1.5-low
    size: 1200x800
    format: webp
    max_candidates: 2
  budget:
    monthly_limit: 100  # 枚
    cost_alert: 5.00  # USD
  quality_gate:
    min_score: 18  # /25 (benchmark-design.md の評価軸合計)
    auto_retry: true
    max_retries: 3
```

### コスト見積（対象プロジェクト）

| シナリオ | 記事/月 | サムネ | 差込 | 月額コスト |
|---------|--------|--------|------|----------|
| ライト | 4記事 | 4枚 (Standard) | 8枚 (Fast) | $0.32 |
| レギュラー | 12記事 | 12枚 (Standard) | 24枚 (Fast) | $0.96 |
| ヘビー | 30記事 | 30枚 (Standard) | 60枚 (Fast) | $2.40 |
| 量産 | 100記事 | 100枚 (Fast Batch) | 200枚 (Fast Batch) | $3.00 |

### 品質保証フロー

```
[生成画像]
    |
    v
[Claude Vision 評価]
    |
    ├── スコア >= 18/25 → 採用
    |
    ├── スコア 15-17/25 → プロンプト微調整して再生成 (最大3回)
    |
    └── スコア < 15/25 → モデル変更して再生成
```

## 6. 横断的な考慮事項

### 6.1 API Key管理

全プロジェクト共通で以下のAPI Key管理方針を適用:

- API Keyは環境変数で管理。`.env` ファイルは `.gitignore` に含める
- MCP設定では `${ENV_VAR}` 形式で環境変数参照
- harness-harness の private/ にAPI Key情報は格納しない
- プロジェクト間でAPI Keyは共有しない（課金管理のため）

### 6.2 コスト監視

- 各プロジェクトの月間画像生成コストを追跡
- 設定した月額上限を超えそうな場合は警告
- Batch API の活用を自動提案（月間50枚以上の場合）

### 6.3 法規制対応の自動化

- create-harness / init-project で AI生成画像の開示ルールを CLAUDE.md に自動追記
- 生成画像のメタデータ（SynthID, C2PA）を削除しない方針をデフォルト化
- diagnose-harness で法規制対応状況をチェック項目に含める
