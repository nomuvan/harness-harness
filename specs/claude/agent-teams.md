# Claude Code Agent Teams 仕様

最終更新: 2026-03-24

複数のClaude Codeインスタンスを協調動作させる実験的機能。サブエージェント（結果を返すだけ）と異なり、チームメイトは独立したコンテキストウィンドウを持ち、相互にメッセージングできる。

公式ドキュメント: https://code.claude.com/docs/en/agent-teams

---

## 1. 有効化

```json
// .claude/settings.json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  },
  "teammateMode": "in-process"
}
```

または環境変数: `export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`

## 2. アーキテクチャ

| コンポーネント | 役割 |
|---------------|------|
| リード | チームを作成・統括するメインセッション |
| チームメイト | 独立したClaude Codeインスタンス。各自1Mトークンのコンテキストウィンドウ |
| 共有タスクリスト | `~/.claude/tasks/{team-name}/` に保存。依存関係サポート |
| メールボックス | ピアツーピアメッセージング |

## 3. API

### TeamCreate
- リードセッションがチームを作成
- チームメイトからは呼び出し不可

### SendMessage
- メッセージ種別:
  - `message`: 特定チームメイトへの直接メッセージ
  - `broadcast`: 全チームメイトへ一斉送信
  - `shutdown_request`: グレースフルシャットダウン要求
  - `shutdown_response`: シャットダウン承認/拒否
  - `plan_approval_response`: 計画のクオリティゲート

### TeamDelete
- 全チームメイトシャットダウン後にのみ呼び出し可能
- リードが実行

## 4. 表示モード

| モード | 設定値 | 説明 |
|--------|--------|------|
| in-process | `"in-process"` | メインターミナル内。Shift+Downでチームメイト間を切り替え |
| tmux分割 | `"tmux"` | 各チームメイトが独立ペイン。全出力を同時表示 |
| 自動 | `"auto"` | tmux内ならsplit、それ以外はin-process |

## 5. 計画承認（Plan Approval）

複雑・リスクのあるタスクでは、チームメイトに計画承認を要求できる:

```
Spawn an architect teammate to refactor the authentication module.
Require plan approval before they make any changes.
```

- チームメイトは読み取り専用のplanモードで作業
- 計画完了後、リードに承認リクエストを送信
- リードが承認 → 実装開始。却下 → フィードバック付きで再計画
- リードの判断基準はプロンプトで制御可能（「テストカバレッジを含む計画のみ承認」等）

## 6. Hooksとの連携（品質ゲート）

Agent Teamsは以下のhookイベントで品質を強制できる:

- **`TeammateIdle`**: チームメイトがidle状態になるとき実行。exit code 2でフィードバックを送りて作業を継続させる
- **`TaskCompleted`**: タスクが完了マークされるとき実行。exit code 2で完了を阻止しフィードバックを送る

## 7. チームメイト定義（.claude/agents/*.md）

`.claude/agents/` にMarkdownファイルとして定義:
- モデル選択（haiku, sonnet, opus）
- 利用可能ツール
- 実行フェーズ（初期化→作業ループ→シャットダウン）
- エラーハンドリング戦略

## 8. チームメイトへの直接操作

### in-processモード
- `Shift+Down`: チームメイト間を順に切り替え（最後のチームメイトの次はリードに戻る）
- `Enter`: チームメイトのセッションを表示
- `Escape`: チームメイトの現在のターンを中断
- `Ctrl+T`: タスクリストの表示/非表示切り替え
- テキスト入力で直接メッセージ送信

### split-paneモード
- 各チームメイトのペインをクリックして直接操作

## 9. コンテキストと権限

- チームメイトはリードの会話履歴を**引き継がない**。スポーン時のプロンプトのみ受け取る
- CLAUDE.md、MCPサーバー、Skillsは通常セッションと同様に読み込まれる
- 権限はリードの設定を引き継ぐ。スポーン後に個別変更可能だが、スポーン時に個別設定はできない

## 10. サブエージェント vs Agent Teams

| 観点 | サブエージェント | Agent Teams |
|------|----------------|-------------|
| 実行モデル | 単一セッション内。結果をcallerに返す | 独立インスタンス。完全自律 |
| 通信 | 結果の返却のみ | ピアツーピアメッセージング |
| 協調 | ハブ&スポーク（メインが管理） | 自己協調（共有タスクリスト） |
| コンテキスト | 結果が要約されて戻る | 各自独立コンテキスト。メッセージで共有 |
| 適用場面 | 集中並列タスク（調査、検証） | 協調が必要な作業（議論、交渉） |
| トークンコスト | 低い | 高い（3-4倍） |
| セットアップ | シンプル（Agent tool） | 実験的フラグ必要。オーバーヘッド大 |

### 判断フロー

```
調査・読み取りのみ？ → サブエージェント（Explore）
単一モジュール変更？ → サブエージェント（General-Purpose）
複数モジュール並列変更？ → Agent Teams
複雑要件（スコープクリープリスク）？ → Agent Teams + リマインダーエージェント
判断に迷う？ → サブエージェント（シンプルな方）
```

## 11. ベストプラクティス

### チームサイズ
- 最適: 3-5チームメイト
- 5-6を超えると協調オーバーヘッドが利点を上回る

### タスク設計
- チームメイトごと5-6タスク
- 自己完結型で明確な成果物
- **ファイル所有権を分離**（同一ファイルの同時編集は上書きによるデータロスの原因）

### リマインダーエージェントパターン
- Haikuモデルで低コスト運用
- 約60秒間隔でリードに要件・計画をリマインド
- スコープクリープ・品質低下を防止
- パッシブ受信（リードは読むが返信不要）

### 注意事項
- `/resume`でin-processチームメイトは復元不可
- タスクステータスが更新されないことがある（手動確認推奨）
- チームメイトのシャットダウンに時間がかかる場合がある
- ネストされたチームは不可（チームメイトがチームを作ることは不可）

## 12. 強力なユースケース

### 並列コードレビュー
```
3人のレビュアー: セキュリティ担当、パフォーマンス担当、テストカバレッジ担当
```

### 競合仮説デバッグ
```
5人の調査員が異なる仮説を並列検証。相互に議論して仮説を反証
```

### クロスレイヤー機能開発
```
API担当、DBマイグレーション担当、テスト担当が並列作業
```

## 13. 制限事項（公式）

- `/resume`、`/rewind`でin-processチームメイトは復元不可
- タスクステータスが更新されない場合がある（依存タスクがブロックされる）
- チームメイトのシャットダウンに時間がかかる（現在のリクエスト/ツール完了まで待機）
- 1セッション1チームのみ（前のチームをクリーンアップしてから新チーム作成）
- ネストされたチーム不可（チームメイトがチーム作成不可）
- リード固定（チームメイトをリードに昇格不可）
- 権限はスポーン時にリードから引き継ぎ（個別設定は後から変更可能だがスポーン時は不可）
- split-paneはVS Code統合ターミナル、Windows Terminal、Ghosttyでは非サポート

## 14. Codex CLIとの比較

Codex CLIにはAgent Teamsの直接的な対応機能は**存在しない**。
Codexの`multi_agent`フィーチャーフラグはあるが、Claude CodeのAgent Teamsとは異なる実装。

| 機能 | Claude Code | Codex CLI |
|------|------------|-----------|
| マルチエージェント協調 | Agent Teams（実験的） | multi_agent（フィーチャーフラグ、詳細不明） |
| ピアツーピアメッセージ | ✅ | ❌ |
| 共有タスクリスト | ✅ | ❌ |
| チームメイト定義 | .claude/agents/*.md | ❌ |

→ mapping分類: **claude-native**
